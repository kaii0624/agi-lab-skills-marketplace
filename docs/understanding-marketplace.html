AGI Lab Skills Marketplace
構造理解ガイド

この資料は「このリポジトリが何で、どこを触ると何が変わるのか」を 最短で掴むためのガイドです。

対象: 初見ユーザー 主眼: フォルダ構造の意味 使い方: そのまま改造可能
1. 全体像 2. フォルダ構造 3. 各ファイルの意味 4. 収録プラグイン 5. 変更時の流れ

1. まず全体像

Marketplace 定義 → Plugin 定義 → Skill 定義 → Script 実行

ざっくり言うと、.claude-plugin/marketplace.json が「棚」、 plugins/*/.claude-plugin/plugin.json が「商品ラベル」、 skills/*/SKILL.md が「使い方・振る舞い」、 scripts/* が「実際に動く処理」です。

.claude-plugin/

リポジトリ全体の marketplace 設定。どの plugin を公開するかを定義します。

plugins/

プラグイン本体の置き場。1フォルダ = 1プラグインです。

skills/

エージェントの振る舞い定義。呼び出し条件や出力ルールが入ります。

scripts/

必要なら実行ファイルを置く層。terminal-vibes はここを呼び出して演出します。

3. 重要ファイルの意味

ファイル 何を決めるか ここを変えると何が変わるか
.claude-plugin/marketplace.json marketplace 名、公開される plugin 一覧 インストール時に見えるマーケット情報と配布対象が変わる
plugins/*/.claude-plugin/plugin.json plugin 名、説明、version、author plugin 一覧表示の見え方が変わる
plugins/*/skills/*/SKILL.md 呼び出し条件、振る舞い、出力フォーマット エージェントが実際にどう動くかが変わる
plugins/terminal-vibes/scripts/* ASCII 表示などの実行処理 画面演出の内容や時間が変わる
README.md 利用者向け入口、手順、デモ導線 初見の理解速度と試されやすさが変わる

4. 収録プラグインを一言で

hackathon-starter

提出用の土台。
自分のアイデアを「対象ユーザー・入力・出力・自律ポイント」に整理して README とデモまで繋げるためのテンプレートです。

実運用用というより、最初の設計ガイドとして使う想定です。

terminal-vibes

遊び用プラグイン。
ASCII ネコ、ダジャレ、ドーナツ、Matrix を実行して休憩体験を提供します。

cat と joke はその場実行、donut と matrix は別ターミナル案内が基本です。

5. 何か変更したいときの順番

  1. まず README.md で目的を明文化する
  2. skills/*/SKILL.md で入力・出力・ルールを固定する
  3. 必要なら scripts/* を実装する
  4. plugin.json の説明と version を更新する
  5. 最後に marketplace.json の plugin 一覧を確認する

迷ったら「README と SKILL.md だけで使い方が伝わるか」を先に満たすと失敗しにくいです。
